## 현재 판단

- Container Manager가 설치되어 있으므로 컨테이너 기반 배포를 우선 후보로 삼을 수 있다.
- x86 계열 CPU이므로 `amd64` 컨테이너 이미지를 우선 검토한다.
- 1 GB 메모리는 핵심 제약이므로 여러 상시 컨테이너와 메모리 사용량이 큰 데이터베이스를 초기 구조의 기본값으로 삼지 않는다.
- 단일 애플리케이션 컨테이너와 경량 데이터 저장소 구성을 우선 검증한다.
- 컨테이너별 CPU·메모리 제한과 실제 DSM 여유 메모리를 측정한 후 운영 한도를 확정한다.
- 별도 서비스를 거의 사용하지 않는 기준 상태에서 메모리 사용률은 약 53%다. 파일 캐시를 포함한 DSM의 메모리 표시 특성을 고려하여 사용률만으로 가용 메모리를 단정하지 않는다.
- 실제 배포 가능성과 성능은 Elaboration 단계의 시험 배포로 검증한다.
- RAID 1로 추정되지만 별도 외부 백업이 없어 NAS 고장·분실·랜섬웨어·운영 실수에 대한 복구 수단이 없다.

## Elaboration 검증 항목

- 대상 CPU 아키텍처에서 선택한 컨테이너 이미지가 실행되는가
- DSM 기본 서비스가 동작하는 상태에서 애플리케이션의 유휴·부하 메모리가 허용 범위인가
- 메모리 부족으로 스왑 사용, 서비스 종료 또는 DSM 응답 저하가 발생하지 않는가
- 정상 사용과 24시간 유휴 상태에서 메모리 사용량이 계속 증가하지 않는가
- NAS 재시작 후 서비스가 자동 복구되는가
- 초기 화면 및 API 응답시간이 품질 목표를 충족하는가
- 데이터와 설정이 컨테이너 재생성 후에도 유지되는가
- 백업한 데이터로 복구할 수 있는가

### TV 검증 항목

- 로컬 HTTPS 인증서를 오류 없이 처리하는가
- 6자리 기기 연결 코드와 한글 UI를 정상 표시하는가
- 3840×2160 화면에서 글자 크기와 정보 밀도가 시청 거리에 적합한가
- 약 3 m 거리에서 가족 구성원이 핵심 정보와 경고를 확대 조작 없이 읽을 수 있는가
- 브라우저 재실행 후 TV 기기 인증정보가 유지되는가
- 일정·날씨의 주기적 HTTP 갱신과 필요한 경우 실시간 갱신이 동작하는가
- 변경 반영이 일정·할 일 60초, 중요 공지 10초 품질 목표를 충족하는가
- 24시간 장시간 표시 중 메모리 증가, 멈춤 또는 새로고침 실패가 없는가
- TV 화면 보호기와 자동 절전이 대시보드 사용 목적에 맞게 설정 가능한가
- 내장 브라우저가 부적합하면 HDMI 외부 출력 장치로 전환 가능한가
- 전용 앱이 필요하면 Tizen Web `.wgt` 패키지로 기존 웹 코드를 재사용할 수 있는가
