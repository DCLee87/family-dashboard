# Family Dashboard 최초 설정 가이드

> 상태: 구현 전 초안
>
> 이 문서는 합의된 운영 흐름을 설명한다. 실제 패키지명, URL, 버튼명과 로그 문구는 Elaboration 시험 배포와 Transition 검증 후 확정한다.

## 목적

시놀로지 NAS에서 Family Dashboard를 처음 실행하고 다음 초기 상태를 안전하게 구성한다.

- 최초 신뢰 PC 등록
- 4자리 관리자 PIN 설정
- PIN 복구 코드 보관
- 가족 기본 정보와 집 날씨 지역 설정 준비
- 이후 모바일·TV·태블릿을 등록할 수 있는 기준점 생성

## 준비 사항

- DS216+II와 같은 집 안 네트워크에 연결된 PC
- DSM 관리자 권한
- Container Manager 설치
- 복구 코드를 보관할 종이 또는 신뢰할 수 있는 비밀번호 관리자
- 초기 설정 중 다른 사람에게 화면이나 코드를 공유하지 않을 환경

## 보안 주의사항

- 초기 설정 코드, 관리자 PIN과 복구 코드를 GitHub, 메모 문서, 메신저 또는 스크린샷으로 보관하지 않는다.
- 복구 코드는 NAS 안에만 보관하지 않는다. NAS 장애 시 함께 잃어버릴 수 있다.
- DSM 관리자 화면과 Container Manager를 인터넷에 공개하지 않는다.
- 최초 설정은 반드시 집 안 네트워크에서 진행한다.

## 1. 컨테이너 시작

1. DSM에 관리자 계정으로 로그인한다.
2. Container Manager에서 Family Dashboard 프로젝트를 연다.
3. 데이터·설정·백업 볼륨의 NAS 경로가 올바른지 확인한다.
4. 컨테이너를 시작한다.
5. 상태가 `실행 중`인지 확인한다.

예정 데이터 경계:

```text
/docker/family-dashboard/
├─ data/
├─ config/
├─ backups/
└─ logs/
```

## 2. 일회용 초기 설정 코드 확인

1. Container Manager에서 Family Dashboard 컨테이너의 로그를 연다.
2. `Initial setup code`에 해당하는 일회용 코드를 확인한다.
3. 코드를 다른 사람과 공유하거나 장기 보관하지 않는다.
4. 코드가 보이지 않거나 이미 사용됐다는 메시지가 나오면 임의로 초기화하지 말고 운영 복구 절차를 확인한다.

구현 시 코드의 정확한 형식과 로그 문구를 이 문서에 반영한다.

## 3. 최초 설정 화면 접속

1. 같은 집 안 네트워크의 PC에서 Family Dashboard 로컬 HTTPS 주소를 연다.
2. 최초 설정 화면이 표시되는지 확인한다.
3. 외부 네트워크에서는 이 화면이 열리지 않아야 한다.
4. 초기 설정 코드를 입력한다.
5. 여러 번 실패하면 잠시 입력이 제한될 수 있다.

예정 주소 형식:

```text
https://<family-dashboard-local-address>/setup
```

## 4. 관리자 PIN 설정

1. 사용할 4자리 숫자 PIN을 입력한다.
2. 같은 PIN을 다시 입력해 확인한다.
3. 생년월일, 전화번호 끝자리, `0000`, `1234`처럼 추측하기 쉬운 값은 피한다.
4. PIN 원문은 서버에 저장되지 않는다는 안내를 확인한다.

정책:

- 5회 연속 실패 시 해당 기기에서 5분간 입력 차단
- 관리자 모드는 마지막 조작 10분 후 자동 잠금
- 화면에서 즉시 잠금 가능

## 5. PIN 복구 코드 보관

1. 화면에 한 번 표시되는 복구 코드를 확인한다.
2. 종이에 기록하거나 신뢰할 수 있는 비밀번호 관리자에 저장한다.
3. NAS 데이터 폴더, Git 저장소 또는 일반 메모 앱에 저장하지 않는다.
4. `복구 코드를 안전하게 보관했습니다` 확인을 선택한다.
5. 복구 코드를 잃어버렸다면 설정을 완료하기 전에 새 코드를 발급받는다.

복구 코드는 등록된 PC에서 PIN을 재설정할 때 사용하며, 사용 후 무효화되고 새 코드가 발급된다.

## 6. 최초 신뢰 PC 등록

1. PC 기기 이름을 입력한다. 예: `거실 데스크톱`.
2. 최초 신뢰 PC 등록을 확인한다.
3. 서버가 이 PC 전용 인증정보를 발급한다.
4. 등록 완료 메시지를 확인한다.
5. 초기 설정 코드는 즉시 무효화되고 `/setup` 화면은 닫혀야 한다.

## 7. 초기 설정 완료 확인

다음 항목을 확인한다.

- [ ] 같은 초기 설정 코드를 다시 사용할 수 없음
- [ ] 최초 신뢰 PC가 기기 목록에 표시됨
- [ ] 기본 상태가 보기 모드임
- [ ] 4자리 PIN으로 관리자 모드 잠금 해제 가능
- [ ] 즉시 잠금과 10분 자동 잠금이 동작함
- [ ] 복구 코드가 NAS 외부에 안전하게 보관됨
- [ ] 외부 미등록 기기에서 가족 데이터가 보이지 않음
- [ ] DSM 관리자 화면은 인터넷에 공개되지 않음

## 8. 다음 설정 순서

최초 PC 등록 후 다음 순서로 설정한다.

1. 가족 구성원: 아빠, 엄마, 딸
2. 기본 집 장소와 시·군·구 날씨 지역
3. 부모 모바일 등록
4. 공용 태블릿 등록
5. 거실 TV 등록
6. 외부 접속과 HTTPS 검증
7. 부모 모바일 Web Push 시험
8. 백업과 복구 시험

## 문제 발생 시 원칙

- 초기 설정을 반복하기 위해 데이터 폴더를 임의 삭제하지 않는다.
- 컨테이너를 재생성하기 전에 데이터·설정 볼륨이 유지되는지 확인한다.
- PIN만 잊어버렸다면 등록된 PC와 복구 코드를 사용한다.
- 최초 PC와 복구 코드를 모두 잃어버린 경우에는 구현 후 제공될 NAS 운영자 복구 절차를 사용한다.
- 운영자 복구 절차가 검증되기 전에는 실제 가족 데이터를 유일본으로 저장하지 않는다.

## Transition 단계 검증 기록

실제 배포 시 아래를 기록한다.

```text
검증일:
DSM 버전:
애플리케이션 버전:
설정 수행 기기:
초기 설정 성공 여부:
자동 잠금 검증:
외부 미등록 기기 차단 검증:
복구 코드 재설정 시험:
발견된 문제:
```
