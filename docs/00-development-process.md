# 개발 프로세스

## 적용 방법

프로젝트는 Unified Process(UP)를 적용한다. 각 단계 안에서 요구사항, 분석, 설계, 구현 및 검증을 반복하며 위험도가 높은 항목을 먼저 다룬다.

단계별 산출물과 완료 상태는 `docs/up/` 아래에서 관리한다. 공통 요구사항 문서는 한 곳에만 두고 각 단계의 인덱스에서 연결하여 중복 문서가 서로 달라지는 것을 방지한다.

## 단계

### 1. Inception

목표는 프로젝트의 가치와 범위, 이해관계자, 핵심 유스케이스, 주요 품질 목표와 위험을 합의하는 것이다.

주요 산출물:

- 비전과 범위
- 이해관계자 및 기기·권한
- 핵심 기능 요구사항
- 품질 요구사항 초안
- 핵심 유스케이스 목록
- 위험 목록
- 초기 용어집

종료 조건:

- 초기 버전의 범위와 제외 범위가 합의됨
- 핵심 사용자와 권한이 합의됨
- 상위 위험 및 품질 목표가 식별됨
- Elaboration에서 검증할 항목이 정해짐

### 2. Elaboration

목표는 핵심 유스케이스를 상세화하고 아키텍처와 주요 위험을 검증하는 것이다.

주요 산출물:

- 상세 유스케이스와 인수 기준
- 아키텍처 기준선
- 데이터 및 보안 설계
- NAS 배포 실험
- 기기별 UI 프로토타입
- 품질 측정 및 테스트 계획

### 3. Construction

우선순위가 정해진 유스케이스를 반복 단위로 구현하고 테스트한다. 각 반복 결과는 실행 가능해야 한다.

### 4. Transition

시놀로지 NAS에 배포하고 실제 TV, PC, 태블릿 및 모바일에서 사용자 검증을 수행한다. 백업, 복구 및 운영 절차도 확정한다.

## 반복 원칙

- 각 반복은 목표와 완료 조건을 가진다.
- 요구사항 변경은 관련 `FR`, `QR`, `UC`, `AC`, `RSK`, `ADR`에 반영한다.
- 중요한 기술 선택은 ADR로 이유와 결과를 기록한다.
- 품질 요구사항은 가능한 경우 측정 가능한 인수 기준으로 작성한다.
- 단계 산출물이 새로 생기거나 상태가 바뀌면 해당 `docs/up/<단계>/README.md`를 함께 갱신한다.

## 품질 모델

- 제품 품질 요구사항은 ISO/IEC 25010:2023을 참조한다.
- 사용 맥락에서의 품질이 필요하면 ISO/IEC 25019:2023을 보조적으로 참조한다.
- 표준 문구를 복제하기보다 이 프로젝트에 필요한 품질 목표와 측정 방법으로 구체화한다.
