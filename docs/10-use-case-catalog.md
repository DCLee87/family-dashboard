# 핵심 유스케이스 목록

## 목적

Inception 단계에서 가족이 시스템을 통해 달성하려는 목표를 식별한다. 상세 정상 흐름, 대안 흐름 및 인수 기준은 우선순위가 확정된 후 작성한다.

## 액터

| 액터 | 설명 |
|---|---|
| 가족 보기 사용자 | 부모 또는 딸이 보기 모드에서 정보를 확인함 |
| 관리자 | PC·부모 모바일·공용 태블릿에서 PIN으로 관리자 모드를 잠금 해제한 사용자 |
| 거실 TV | 항상 보기 전용인 공용 대시보드 기기 |
| 등록된 부모 모바일 | Web Push를 받을 수 있는 아빠 또는 엄마의 기기 |
| 예약 작업 | 반복 회차, 알림, 날씨 및 자동 삭제를 처리하는 시스템 작업 |
| 외부 날씨 서비스 | 설정된 지역의 날씨 데이터를 제공하는 외부 시스템 |

## 우선순위 기준

- `Must`: 초기 버전의 핵심 가치에 반드시 필요
- `Should`: 초기 버전에 포함하는 것이 중요하지만 핵심 흐름을 막지는 않음
- `Could`: 시간과 위험 검증 결과에 따라 포함
