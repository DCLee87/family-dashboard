# 품질 요구사항

## 기준

제품 품질 모델은 ISO/IEC 25010:2023을 참조한다. 아래 수치는 Inception 단계의 초안이며 사용자 합의와 NAS 실측을 거쳐 확정한다.

| ID | 품질 영역 | 요구사항 및 검증 방향 | 상태 |
|---|---|---|---|
| QR-FS-001 | 기능 적합성 | 일정 시간과 장소로 계산한 가족 상태가 정의된 규칙과 일치해야 한다. 규칙 기반 테스트로 검증한다. | 초안 |
| QR-PE-001 | 성능 효율성 | 가정 내 네트워크에서 TV 첫 화면이 목표 시간 안에 표시되어야 한다. 목표 시간은 Elaboration에서 NAS 실측 후 확정한다. | 수치 미정 |
| QR-PE-002 | 성능 효율성 | 화면 자동 갱신이 TV 시청을 방해하는 전체 깜박임을 일으키지 않아야 한다. | 초안 |
| QR-PE-003 | 성능 효율성 | DS216+II 시험 배포에서 애플리케이션 컨테이너의 정상 사용 메모리를 우선 256 MB 이내로 목표한다. 실제 측정 후 확정한다. | 목표 수치 초안 |
| QR-PE-004 | 성능 효율성 | 24시간 유휴·정상 사용 시험에서 지속적인 메모리 증가, 비정상 종료 또는 DSM의 현저한 응답 저하가 없어야 한다. | 초안 |
| QR-CO-001 | 호환성 | 대상 TV 브라우저, PC, iOS, Android 및 태블릿에서 핵심 조회 기능이 동작해야 한다. 구체적인 최소 버전은 기기 조사 후 확정한다. | 범위 미정 |
| QR-UI-001 | 상호작용성 | TV의 핵심 정보는 일반 시청 거리에서 식별 가능해야 한다. 실제 TV와 거리 조건으로 검증한다. | 조건 미정 |
| QR-UI-002 | 상호작용성 | 8세 보기 사용자가 편집 기능에 들어가지 않고 오늘의 일정과 할 일을 확인할 수 있어야 한다. 사용자 시나리오로 검증한다. | 초안 |
| QR-RE-001 | 신뢰성 | NAS 또는 애플리케이션 재시작 후 수동 데이터 복구 없이 정상 서비스로 복귀해야 한다. | 초안 |
| QR-RE-002 | 신뢰성 | 외부 날씨 서비스가 실패해도 일정·할 일·공지 조회가 가능해야 한다. | 초안 |
| QR-RE-003 | 신뢰성 | 휴지통 보관 기간 안의 삭제 일정은 원래 내용과 대상 가족, 반복 및 공개 범위를 유지한 채 복원할 수 있어야 한다. | 필수 |
| QR-RE-004 | 신뢰성 | Push 발송 실패가 일정·할 일·공지 조회 및 편집 기능의 장애를 일으켜서는 안 된다. | 필수 |
| QR-RE-005 | 신뢰성 | 알림 예약 계산은 시스템 기본 시간대와 반복 일정 예외를 일관되게 적용해야 한다. | 필수 |
| QR-SE-001 | 보안성 | PIN과 비밀번호를 원문으로 저장하지 않아야 한다. | 필수 |
| QR-SE-002 | 보안성 | 보기 전용 기기에서는 데이터 변경 API가 인증 없이 성공해서는 안 된다. | 필수 |
| QR-SE-003 | 보안성 | 비밀값과 실제 주소는 Git 저장소에 저장하지 않아야 한다. | 필수 |
| QR-SE-004 | 보안성 | TV와 권한 없는 화면의 API 응답에는 숨겨야 하는 일정 제목, 장소 및 상세 내용이 포함되어서는 안 된다. 화면에서만 CSS로 숨기는 방식은 허용하지 않는다. | 필수 |
| QR-SE-005 | 보안성 | 일정 삭제, 휴지통 복원 및 영구 삭제 작업은 유효한 관리자 모드를 검증해야 한다. | 필수 |
| QR-SE-006 | 보안성 | 외부 접속에서 관리자 PIN만으로 사용자 인증을 대체해서는 안 된다. | 필수 |
| QR-SE-007 | 보안성 | 4자리 PIN의 추측 위험을 줄이기 위해 연속 실패 횟수 제한과 시간 기반 입력 차단을 서버에서 적용해야 한다. | 필수 |
| QR-SE-008 | 보안성 | PIN 복구 코드 원문을 저장하지 않고 등록된 PC 세션과 유효한 복구 코드를 서버에서 함께 검증해야 한다. | 필수 |

| QR-SE-010 | 보안성 | Push 구독 endpoint와 암호화 키는 비밀정보로 취급하고 권한 없는 화면과 로그에 노출하지 않아야 한다. | 필수 |
| QR-MA-001 | 유지보수성 | 일정, 할 일, 공지, 날씨 및 가족 상태 위젯을 독립적으로 변경·시험할 수 있는 구조를 지향한다. | 초안 |
| QR-FL-001 | 유연성 | 화면 구성은 소스코드 수정 없이 기기 프로필별 표시 여부와 순서를 변경할 수 있어야 한다. | 필수 |
| QR-SA-001 | 안전성 | 일정 기반 상태는 실제 위치로 표시해서는 안 되며 `일정 기준`과 유효 종료 시각을 함께 제공해야 한다. | 필수 |
| QR-SA-002 | 안전성 | 겹치는 일정으로 하나의 예상 상태를 선택한 경우 사용자가 다른 일정의 존재를 알 수 있도록 겹침 표시를 제공해야 한다. | 필수 |
| QR-SA-003 | 안전성 | 부모 전용 일정의 비식별 상태는 `개인 일정`과 종료 시각만 포함하고 실제 위치나 일정 상세를 노출하지 않아야 한다. | 필수 |
| QR-SA-004 | 안전성 | 종일 일정은 명시적인 상태 반영 시간대가 없으면 가족의 예상 위치 상태로 사용해서는 안 된다. | 필수 |

## 추적 원칙

- 각 품질 요구사항은 하나 이상의 테스트 또는 검토 항목과 연결한다.
- `수치 미정` 항목은 Elaboration 종료 전에 측정 가능한 인수 기준으로 바꾼다.
- 기술 제약 때문에 목표를 바꾸면 위험 목록과 ADR을 함께 갱신한다.
