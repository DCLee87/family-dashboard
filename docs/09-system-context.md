# 시스템 구성

## 기준 아키텍처 후보

- 클라이언트는 PC, TV, iOS, Android 및 태블릿에서 사용하는 적응형 PWA다.
- 거실 TV는 보기 전용 기기 프로필을 사용한다.
- 시놀로지 DS216+II의 Container Manager에서 경량 애플리케이션 컨테이너 하나를 우선 사용한다.
- 애플리케이션 컨테이너는 웹 화면, API, 일정·상태 계산 및 예약 작업을 포함한다.
- 초기 데이터 저장소는 별도 데이터베이스 컨테이너보다 SQLite를 우선 검증한다.
- 데이터, 설정 및 백업은 컨테이너 외부 NAS 볼륨에 저장한다.
- Reverse Proxy와 HTTPS를 통해 PWA와 서비스 워커의 보안 컨텍스트를 제공한다.

## 외부 시스템

| 외부 대상 | 목적 | 장애 시 원칙 |
|---|---|---|
| 날씨 API | 지역별 현재·예보 날씨 | 기존 캐시 또는 오류 상태 표시, 핵심 기능 유지 |
| 브라우저 Push 서비스 | 부모 모바일 Web Push 전달 | 실패 기록, 일정·할 일 등 핵심 기능 유지 |
| 외부 접속 계층 | 집 밖에서 안전하게 접속 | 구체적 방식은 Elaboration에서 결정 |

## 배포 데이터 경계

```text
/docker/family-dashboard/
├─ data/       # SQLite와 영속 데이터
├─ config/     # 환경설정과 공개되지 않는 키
├─ backups/    # 백업 파일
└─ logs/       # 제한적으로 보관하는 운영 로그
```

## Elaboration 검증 항목

- DS216+II에서 단일 컨테이너와 SQLite의 메모리·응답시간
- 실제 거실 TV 브라우저의 장시간 표시와 자동 갱신
- 부모 iOS·Android 기기의 PWA 설치 및 Web Push 수신
- HTTPS 인증서와 내부·외부 접속 방식
- NAS 재시작 후 자동 복구와 영속 데이터 유지
