# 위험 목록

가능성과 영향은 `낮음`, `중간`, `높음`의 초기 추정치다.

| ID | 위험 | 가능성 | 영향 | 대응 및 검증 계획 | 상태 |
|---|---|---:|---:|---|---|
| RSK-001 | DS216+II의 2코어 CPU와 1 GB 메모리가 목표 구조에 부족함 | 중간 | 높음 | 상시 서비스가 거의 없는 상태에서 메모리 사용률 약 53%. x86/amd64 경량 이미지, 단일 애플리케이션 및 경량 저장소를 우선 검토하고 Elaboration 초기에 메모리·응답시간·재시작 시험 수행 | 기준 측정·검증 필요 |
| RSK-002 | 거실 TV 브라우저가 최신 웹 기능이나 장시간 표시를 안정적으로 지원하지 않음 | 중간 | 높음 | TV 모델·브라우저 확인 후 대시보드 프로토타입 장시간 테스트 | 미확인 |
| RSK-003 | 외부 접속 구성이 가족 데이터를 인터넷에 노출함 | 중간 | 높음 | VPN 또는 신뢰 가능한 접근 방식 비교, 직접 포트 노출 최소화 | 미확인 |
| RSK-004 | 공용 PIN 유출로 딸 또는 방문자가 데이터를 변경함 | 중간 | 중간 | 실패 제한, 자동 잠금, PIN 변경 및 최소 권한 적용 | 대응 예정 |
| RSK-005 | 일정 기반 상태가 실제 위치로 오해됨 | 중간 | 높음 | `일정 기준`, 종료 시각 및 오래된 상태 표시를 필수화 | 대응 정의됨 |

| RSK-007 | 2베이 NAS 장애 또는 디스크 고장으로 가족 데이터가 유실됨 | 낮음 | 높음 | RAID와 백업을 구분하고 외부 백업·복구 테스트 정책 수립 | 미확인 |
| RSK-008 | 요구사항이 늘어나 초기 버전 완성이 지연됨 | 높음 | 중간 | Must/Should/Could 우선순위와 반복별 범위 고정 | 대응 예정 |

2. 거실 TV 모델과 브라우저 동작 특성
3. 내부망·외부망 접속 요구와 보안 방식
4. 백업 대상과 복구 목표
