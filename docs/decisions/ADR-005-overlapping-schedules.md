# ADR-005: 겹치는 일정과 예상 상태 처리

- 상태: 승인
- 결정일: 2026-07-12

## 배경

학교 일정 중 병원에 가는 경우처럼 한 가족 구성원의 일정이 시간상 겹칠 수 있다. 겹치는 일정을 금지하면 실제 생활을 정확히 기록하기 어렵지만, 여러 장소 일정이 동시에 활성화되면 하나의 예상 상태를 선택할 규칙이 필요하다.

## 결정

- 동일한 가족 구성원에게 시간이 겹치는 일정 등록을 허용한다.
- 저장 시 겹치는 기존 일정을 알려주되 사용자가 확인하고 저장할 수 있다.
- 일정 상세 화면에는 겹치는 모든 일정을 표시한다.
- 예상 상태는 현재 활성 상태인 장소 일정 중 시작 시각이 가장 늦은 일정을 우선한다.
- 예상 상태와 대시보드에는 일정 겹침 표시를 제공한다.

## 예시

```text
학교: 09:00~15:00
병원: 14:00~16:00

14:00 이전 예상 상태: 학교
14:00 이후 예상 상태: 병원 · 일정 겹침
```

## 결과

- 실제 생활에서 발생하는 예외 일정을 차단하지 않는다.
- 별도 우선순위 입력 없이 시작 시각으로 일관된 상태를 계산할 수 있다.
- 시작 시각이 같은 일정이 겹치는 경우의 최종 동률 규칙은 Elaboration의 데이터 설계에서 확정해야 한다.
- 일정 겹침이 실제 위치를 의미하지 않는다는 표시 원칙은 계속 적용한다.
