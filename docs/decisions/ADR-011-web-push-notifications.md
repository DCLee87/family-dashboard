# ADR-011: 부모 모바일 Web Push

- 상태: 승인, 기술 검증 필요
- 결정일: 2026-07-12

## 배경

가족 일정 알림은 웹앱을 열어 두지 않은 상태에서도 부모 모바일에 전달될 필요가 있다. 네이티브 앱을 만들지 않으므로 표준 Web Push와 서비스 워커를 사용해야 한다.

## 결정

- 부모 모바일 PWA에서 Web Push를 지원한다.
- 사용자가 기기별로 명시적으로 알림을 허용해야 한다.
- 공용 태블릿 Push는 선택 기능이며 기본 비활성화한다.
- TV는 Push 대신 화면에서 다가오는 일정을 강조한다.
- 부모 전용 일정의 Push 본문은 기본적으로 제목과 장소를 숨긴다.
- 문자, 이메일 및 카카오톡 알림은 초기 범위에서 제외한다.
- Push 실패는 일정·할 일·공지 등 핵심 기능과 격리한다.

## 결과

- iOS에서는 홈 화면에 설치한 PWA와 HTTPS가 필요하다.
- NAS는 예약 시각에 켜져 있고 외부 Push 서비스로 통신할 수 있어야 한다.
- 기기별 Push 구독정보, VAPID 키, 서비스 워커 및 발송 작업이 필요하다.
- 실제 부모 기기의 백그라운드 수신을 Elaboration에서 검증해야 한다.
