# ADR-016: 공지와 메모의 통합 게시 항목

- 상태: 승인
- 결정일: 2026-07-12

## 배경

공지와 메모는 모두 가족 대시보드에 글을 게시하는 기능이며 제목, 내용, 게시 기간, 공개 범위와 같은 공통 속성이 많다. 별도 기능으로 만들면 편집·권한·보관·삭제 흐름이 중복된다.

## 결정

- 공지와 메모를 하나의 `게시 항목` 모델로 관리하고 종류로 구분한다.
- 게시 항목은 제목, 내용, 종류, 중요도, 시작일, 선택적 종료일 및 공개 범위를 가진다.
- 중요도는 `보통`과 `중요`를 지원한다.
- 공개 범위는 일정과 동일하게 `가족 공개`, `TV 요약`, `부모 전용`을 사용하고 기본값은 `가족 공개`다.
- 종료일이 지나면 자동으로 보관하고 관리자는 다시 게시할 수 있다.
- 종료일이 없으면 관리자가 보관하거나 삭제하기 전까지 게시한다.
- TV에서는 게시 중인 중요 공지를 다른 게시 항목보다 우선 표시한다.
- 삭제된 게시 항목은 30일간 휴지통에 보관한다.
- 중요 공지를 게시하면 부모 두 명의 알림 허용 모바일에 Web Push를 보낸다.
- 부모 전용 중요 공지의 Push는 제목과 내용을 숨기고 비식별 문구만 표시한다.
- 가족 공개 중요 공지는 TV와 공용 태블릿 상단에 고정한다.
- 일반 공지와 메모는 기본적으로 Push를 보내지 않는다.
- 사용자별 읽음·확인 추적은 초기 범위에서 제외한다.

## 결과

- 하나의 데이터와 관리 흐름으로 공지·메모를 처리할 수 있다.
- 기기 프로필은 공지와 메모의 표시 우선순위와 개수를 다르게 설정할 수 있다.
- 부모 전용 항목은 서버 응답 단계에서 권한에 맞게 제거해야 한다.
- 사용자별 읽음 확인이 나중에 필요하면 부모별 인증과 딸의 확인 수단을 함께 설계해야 한다.
