# ADR-003: 일정별 복수 대상 가족 지정

- 상태: 승인
- 결정일: 2026-07-12

## 배경

가족 일정에는 한 사람만 참여하는 일정과 여러 사람이 함께 참여하는 일정이 모두 존재한다. 개인 일정과 가족 일정을 별도 종류로 만들면 동일한 일정 기능에 서로 다른 처리 규칙이 생기고, 부모 두 명만 참여하는 일정처럼 중간 형태를 표현하기 불편하다.

## 결정

- 모든 일정에는 대상 가족을 1명 이상 지정한다.
- 하나의 일정에 여러 가족 구성원을 함께 지정할 수 있다.
- 개인 일정과 가족 일정을 별도 데이터 유형으로 나누지 않는다.
- 지정된 대상 가족의 수와 조합으로 개인 또는 공동 일정을 표현한다.
- 일정은 지정된 모든 가족의 일정 조회와 일정 기반 상태 계산에 반영한다.

## 예시

```text
딸 학원   → 딸
아빠 회사 → 아빠
부모 모임 → 아빠 + 엄마
가족 외식 → 아빠 + 엄마 + 딸
```

## 결과

- 하나의 일정 모델로 개인·부모 공동·가족 전체 일정을 표현할 수 있다.
- 일정과 가족 구성원은 다대다 관계가 된다.
- 화면에서는 가족별 색상, 이름 또는 아이콘으로 대상자를 구분해야 한다.
- 여러 대상자의 일정 기반 상태가 동일한 일정에서 계산될 수 있다.
