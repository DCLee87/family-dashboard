# ADR-017: 등록 장소 기반 날씨 지역

- 상태: 승인
- 결정일: 2026-07-13

## 배경

GPS를 수집하지 않으므로 가족이 있는 지역의 날씨는 일정 장소를 기반으로 선택해야 한다. 자유 입력된 장소명만으로 날씨를 조회하면 이름이 모호하거나 상세 주소를 저장하게 될 수 있다. 부모 전용 일정의 지역 날씨를 공용 화면에 표시하면 비공개 장소를 추론할 위험도 있다.

## 결정

- 자주 사용하는 장소를 이름과 시·군·구 수준의 날씨 지역으로 등록한다.
- 실제 상세 주소는 날씨 기능을 위해 저장하지 않는다.
- 하나의 기본 집 장소와 지역을 설정한다.
- 일정은 등록 장소를 선택하거나 임시 장소와 선택적 날씨 지역을 입력할 수 있다.
- TV는 집 날씨와 현재 활성 일정에 연결된 허용된 지역 날씨를 표시한다.
- 같은 날씨 지역은 한 번만 표시한다.
- 장소 일정이 없는 가족은 집 지역을 사용한다.
- 부모 전용 일정의 지역은 딸과 공용 화면의 날씨 목록에 사용하지 않는다.
- 외부 날씨 서비스 실패는 핵심 기능과 격리한다.
- 서버는 지역별 현재 날씨, 24시간 시간별 예보 및 7일 일별 예보를 공통으로 수집한다.
- TV는 현재·12시간·5일, 태블릿은 현재·3일, 모바일은 현재·오늘·내일, PC는 전체 범위를 기본 표시한다.
- 관리자는 기기 프로필별 표시 범위를 변경할 수 있다.
- 같은 지역의 날씨 캐시는 기기 간 공유하고 마지막 갱신 시각을 표시한다.
- 현재·시간별 예보는 30분, 일별 예보는 3시간마다 갱신한다.
- 실패 시 기존 캐시를 유지하고 10분부터 점진적으로 재시도한다.
- 6시간 동안 갱신하지 못하면 오래됨 경고를 표시하고 24시간 후에는 예보 값 대신 사용 불가 상태를 표시한다.

## 결과

- 상세 주소 없이 일정과 날씨를 연결할 수 있다.
- 장소와 날씨 지역은 별도 개념으로 관리된다.
- 일정 공개 범위가 날씨 지역 계산과 API 필터에도 적용되어야 한다.
- 날씨 공급자는 별도로 결정해야 한다.
