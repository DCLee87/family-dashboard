# ADR-009: 종일 및 여러 날 일정

- 상태: 승인
- 결정일: 2026-07-12

## 배경

생일, 공휴일, 여행처럼 특정 시간이 없거나 여러 날 계속되는 일정이 필요하다. 그러나 종일 일정의 장소를 하루 24시간 동안 가족의 예상 위치로 사용하면 실제 상황과 다른 정보를 표시할 수 있다.

## 결정

- 시간 일정과 종일 일정을 모두 지원한다.
- 종일 일정은 하나의 날짜 또는 시작일·종료일이 있는 여러 날 일정으로 등록할 수 있다.
- 여러 날 종일 일정은 적용되는 각 날짜에 계속 표시한다.
- 종일 일정은 기본적으로 일정 기반 예상 상태에 사용하지 않는다.
- 종일 일정에 별도의 상태 반영 시작·종료 시각을 지정한 경우에만 해당 시간 동안 예상 상태를 계산한다.
- 공개 범위와 대상 가족 규칙은 일반 일정과 동일하게 적용한다.

## 예시

```text
가족여행
기간: 2026-07-20~2026-07-22
유형: 종일
상태 반영: 사용 안 함
```

## 결과

- 생일, 공휴일, 여행 등 시간 없는 일정을 자연스럽게 표현할 수 있다.
- 날짜 범위의 포함 규칙과 시간대 처리는 Elaboration 데이터 설계에서 명확히 정의해야 한다.
- 상태 반영 시간대를 입력하는 UI가 일반 종일 일정 등록을 복잡하게 만들지 않도록 선택 항목으로 숨겨야 한다.
